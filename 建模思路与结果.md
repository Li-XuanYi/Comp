# B题：纽约约租车调度决策 — 建模思路与结果

## 赛题概述

纽约出租车市场由传统出租车（黄色/绿色）和约租车（FHV）组成。约租车平台希望：
1. 预测布鲁克林区各区域出发的订单量
2. 设计有竞争力的定价方案
3. 优化新增车辆的区域分配
4. 规划3个基地的最优选址

## 问题1：布鲁克林区订单分布规律与逐小时预测

### 建模思路

**目标**：分析布鲁克林61个编号区域的订单出发分布规律，并预测2019年2月1日各区域逐小时订单量。

#### Step 1 — 数据预处理（Node 02）

- 统一黄色/绿色出租车的列名（`tpep_pickup_datetime` → `pickup_datetime` 等）
- 计算行程时长 `duration_min`
- 过滤规则：
  - 时间范围限定在2019年1月
  - 行程时长 0~180分钟
  - 出发区域（PULocationID）必须在布鲁克林61个区域内
  - 黄/绿出租车额外要求：`trip_distance > 0`、`0 < fare_amount ≤ 500`、`0 < total_amount ≤ 1000`

#### Step 2 — 需求面板构建（Node 03）

- 以 `(zone_id, datetime_hour)` 为粒度，聚合黄色和绿色出租车的上车次数
- 生成完整面板：61区域 × 744小时（1月共31天×24小时）= 45,384行
- 衍生字段：`date`、`hour`、`weekday`、`is_weekend`

#### Step 3 — 特征工程（Node 04）

**日历特征**：
| 特征 | 说明 |
|---|---|
| `hour` | 小时 (0-23) |
| `weekday` | 星期 (0-6) |
| `is_weekend` | 是否周末 |
| `is_holiday` | 是否假日（元旦、马丁·路德·金日） |
| `is_morning_peak` | 早高峰 7-9时 |
| `is_evening_peak` | 晚高峰 16-18时 |
| `is_night` | 夜间 22-5时 |
| `hour_sin/cos` | 小时的周期编码 |

**天气特征**（合成模板，因无外部天气数据）：
| 特征 | 构造方式 |
|---|---|
| `temperature` | 33°F + 日间正弦变化 + 周期波动 |
| `precipitation` | 周末早晚高峰概率0.03，其余0 |
| `snow_indicator` | 低温+降水时为1 |
| `wind_speed` | 8 + 2×cos(hour/24) |
| `visibility` | 降水时7，其余10 |

**滞后特征**：
| 特征 | 说明 |
|---|---|
| `lag_24` | 同区域24小时前订单量 |
| `lag_168` | 同区域168小时（一周）前订单量 |
| `rolling_24_mean` | 过去24小时滑动均值 |
| `rolling_168_mean` | 过去168小时滑动均值 |

#### Step 4 — 基线模型（Node 06）

建立3个统计基线用于对比：

| 基线 | 方法 | 验证集MAE |
|---|---|---|
| `zone_hour_mean` | 按(区域,小时)历史均值 | — |
| `zone_weekday_hour_mean` | 按(区域,星期,小时)历史均值 | 1.982 |
| `recent_week_same_hour` | 上周同时刻实际值 | — |

#### Step 5 — 主模型（Node 07）

**模型选择**：`HistGradientBoostingRegressor`（Poisson损失）

选择Poisson损失的原因：订单量为非负计数数据，Poisson回归天然适合。

**超参数**：
- `learning_rate=0.06`
- `max_iter=160`
- `max_leaf_nodes=31`
- `l2_regularization=0.05`

**训练/验证划分**：
- 训练集：1月1日~1月24日
- 验证集：1月25日~1月31日

**融合策略**：GBDT预测与最佳基线（zone_weekday_hour_mean）做凸组合，网格搜索最优权重。

**预测目标日**：2019年2月1日0-23时，滞后特征取1月31日/1月25日的历史值填充。

### 结果

| 指标 | 值 |
|---|---|
| 验证集 MAE | 1.903 |
| 验证集 RMSE | 3.582 |
| 最佳基线 MAE | 1.982 |
| GBDT单独 MAE | — |
| 融合后优于基线 | 是 |

输出：61区域 × 24小时 = 1464条逐小时预测记录

## 问题2：约租车订单定价方案

### 建模思路

**目标**：为附件3中115条FHV约租车订单设计合理的定价，平衡价格竞争力与利润。

#### Step 1 — OD距离/时长估计（Node 08）

FHV数据缺少`trip_distance`和费用字段，需要估算：

1. **有历史样本的OD对**（样本数≥3）：取黄+绿出租车该OD对的中位距离和中位时长
2. **无历史样本的OD对**：用区域质心距离×1.3作为距离估计，除以历史平均速度（中位数约12 mph）得到时长估计

区域质心通过shapefile多边形顶点均值计算。

#### Step 2 — 出租车参考价格模型（Node 09）

**思路**：约租车定价应参考传统出租车价格，在此基础上下调以获取价格优势。

用黄+绿出租车数据训练回归模型，输入特征：
- `distance`（行程距离）
- `duration_min`（行程时长）
- `PULocationID`、`DOLocationID`（起止区域）
- `hour`、`weekday`、`is_weekend`（时间特征）

**模型**：`HistGradientBoostingRegressor`（squared_error损失），学习率0.06，180轮迭代。

训练后对FHV订单预测"出租车等价价格"。

#### Step 3 — 定价公式

```
成本 = c₀ + c_distance × distance + c_time × duration_min
     = 2.75 + 1.15 × distance + 0.18 × duration_min

价格下限 = 成本 × (1 + 最低利润率15%)
竞争价格 = 出租车参考价 × (1 - 折扣率8%)
FHV定价  = max(价格下限, 竞争价格)

# 拼车订单再打9折，但不低于价格下限的95%
```

**参数说明**：
| 参数 | 值 | 含义 |
|---|---|---|
| `c₀` | $2.75 | 起步成本 |
| `c_distance` | $1.15/英里 | 里程成本 |
| `c_time` | $0.18/分钟 | 时间成本 |
| 最低利润率 | 15% | 保底利润 |
| 折扣率 | 8% | 相对出租车的价格优势 |
| 拼车系数 | 0.90 | 拼车订单折扣 |

### 结果

| 指标 | 值 |
|---|---|
| FHV订单数 | 115 |
| 平均FHV定价 | $15.92 |
| 平均估计成本 | — |
| 平均估计利润率 | 56.03% |
| 参考价格模型MAE | — |

## 问题3：区域车辆分配方案（利润最大化）

### 建模思路

**目标**：给定新增车辆数，将其分配到布鲁克林61个区域，使总利润最大化。输出2019年2月1日中午12时的最优分配。

#### 模型：贪心整数分配

**核心思想**：每辆车逐辆分配，每次选择边际利润最大的区域。

**计算逻辑**：
1. 每个区域有预测需求 `predicted_demand_12pm`（来自问题1）
2. 每个区域有平均利润 `avg_profit`（来自问题2的定价结果）
3. 每个区域有服务能力 `capacity = 60 / avg_service_time`（每小时可完成订单数）
4. 逐辆分配：每辆车分配到 `marginal_served × profit` 最大的区域
5. 已服务的需求不能超过预测需求

**车辆数量**：赛题未指定，参数化给出 N=50、N=100、N=200 三套方案。

### 结果

## 问题4：3个基地最优选址

### 建模思路

**目标**：在布鲁克林61个区域内选3个基地，特殊车型从基地出发到上客点，最小化加权派车时间成本。

#### 模型：加权P-中位问题

**目标函数**：
```
min Σᵢ wᵢ × min_j∈Bases { time(i, j) }
```
其中：
- `wᵢ = predicted_demand_12pm × avg_profit`（区域i的需求×利润加权）
- `time(i, j)` 为区域i到基地j的派车时间
- 从61个区域中选3个作为基地

**求解方法**：穷举 C(61,3) = 35,990 种组合，逐一计算加权总派车时间，取最小值。

**对比方案**：
| 方案 | 选址方式 |
|---|---|
| `optimal_p_median` | 穷举最优 |
| `top_demand_3` | 取需求最大的3个区域 |
| `random_3` | 随机选3个区域 |
| `geographic_kmeans_3` | KMeans地理聚类，每簇选中心最近区域 |

### 结果

| 方案 | 基地区域 | 加权派车时间成本 |
|---|---|---|
| **最优P-中位** | **65; 111; 222** | **25,558.48** |
| 需求Top3 | — | — |
| 随机 | — | — |
| KMeans | — | — |

最优基地：
- **区域65**：Downtown Brooklyn/MetroTech
- **区域111**：Green-Wood Cemetery
- **区域222**：Starrett City

## 已知局限与改进方向

1. **天气数据为合成模板**：赛题要求"收集必要数据"，目前用正弦函数模拟，应替换为真实气象数据
2. **FHV样本量小**：仅115条记录，定价模型泛化性受限
3. **新增车辆数未定**：问题3参数化给出3种规模，需更合理假设
4. **OD估计的替代方案**：无历史样本的OD对使用质心距离×1.3，可引入路网距离
5. **基地选址穷举**：C(61,3)可穷举，但若扩展到更多基地或更大区域需改用启发式算法
